For the next RC planed for December 2004, the wish-list is:

  • a functional *nix port
  • document the cons build system
  • improve the fighter unit (attack, move)
  • endian independent file IO operations
  • several missions (with AI and scripting) to showcase the engine capabilities and also player units scripts

Probably I will also include several features that are planned for 0.9.8 version:

  • terrain, ecosystem and cities automatic generation
  • tactical computer
  • virtual file system and zip IO
  • dialog for loading and saving a mission
  • battlegroup scripting interface
  • selection improvement (that takes into account the rendering position rather than ground/layer maps)
  • serialization/deserialization for CSL classes
  • replays (that is going to be *fun*)
  • portable png loading (tweak pngdib or something else)

As for the 0.9.8, this will be the last version of the engine before a “final” release. Probably it will take some 6 months to add all the wanted features above – as a rough estiamation this would be may 2005.

Right now I do R&D to push this engine to another level – 3D.
