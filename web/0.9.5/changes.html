Engine+Architecture features

  • Support for jpg files (using jpeglib v6b)
  • Support for png files – code by tamlin (using pnglib and pngdib lib made by Jason Summers); this is not portable to *nix since pngdib contains Win specific code
  • Restructured the System; that is, all code that contains code for graphics, sound, input, timer, exit, fonts, surfaces and so on are grouped in a folder in Common/System; these “systems” are unrelated, although they duplicate code.
  • Removed Archive concept and introduced Stream concept; there is an Ansi Stream class, which handles file I/O using standard f-calls (fopen, fread/fwrite and so on)
  • Prims16 file now contains code to clip line segments (don’t quite remember where I snatched the code from :])
  • Input system was remade (PPK concept was introduced – Pure Power Key – that is, a key that is used by the engine; the graphic interface now asks for these kind of keys instead of querying specific DirectX or SDL codes from input system). The input system now transforms the PPK_ codes to specific input system key codes
  • Removed InfantryBay and added a generic Bay manager class
  • Added Aircraft class
  • Added Fighter class, based on Aircraft; added fighter weapons and propulsion subsys
  • Added Chopper class, based on Aircraft; added chopper weapons and propulsion subsys
  • Derived Dropship class from Chopper (previously from Static)
  • Layered buildings (see Rion Headquarters static – it looks quite cool although it is useless as a feature)
  • Support for TTF fonts in SDL using SDL_font lib
  • Audio system for SDL is based on SDL_mixer lib
  • Volume is now expressed inside engine as percents (0..100% - 0 is silence, 100% - full volume); this ease the porting between SDL and DirectX
  • Pan is now expressed as percents (-100%..100% - -100% is full left, silence right and so on)
  • Added a WeaponLib, which you may query if a unit can fire another unit using a certain weapon
  • Quick load and save in engine (CSL classes still won’t serialize/deserialize – too lazy)
  • Projectiles and Aircrafts now have a Z axis
  • Added “time-to-live” for projectiles
  • Added code to detect platform (operating system and compiler)
  • Cons (http://www.dsmit.com/cons) script to build on *nix (still to improve, for selection of system – SDL or other, certain help for usage)
  • A particle system (codenamed Mot); it has powerful features but, unfortunately it’s not 100% operational and is not integrated into engine nor it has an editor
  • Terrain generators (will be integrated into 0.9.8 for the automatic map generation feature)
  • Scripting interface for Aircrafts
  • Scripting interface for Statics
  • Aircraft and Dropship Info controls
  • Added Boids class (not integrated/editable in the engine)
  • Added Oscilator subsys
  • Reading and writing structures (statics, infantry, vehicles) is performed through serialization/deserialization interface (and is no longer dependent on enum size or data alignment, allowing flexibility)
  • Alpha blending; PaintSpriteToSurface and PaintSurfaceToSurface now take a new parameter (initialized) - an alpha value; it was tested on SDL and it works; it was also tested on DirectX, but it seems that it won’t work (either with the code or with my hardware)


Tools features

  • Statics editor is able to edit layered buildings (one layer may contain a sprite, which may be animated or not)
  • UnitsEd contains another tab (Visual); for now it contains editable properties when a unit dies
  • Restructured the custom C++Builder components projects (it seems that in this 6th version, Borland made a lot of changes, separating design time and runtime code for components)
  • Added another C++Builder component – a HTTP label that I wanted for some time to add inside “About” boxes(clicks and open a link in the current browser); it does not perform URL checking at design time, but it is usable
  • Added code to handle aircrafts in the object tree (Map Editor)
  • Added code to handle aircrafts and statics parameters in the Object properties (Map Editor)
  • Added code to handle bay for units that may transport other units (a separate window appears where you may be able to add an existing unit or clone a unit from the data files)
  • “Open Recent” item in File menu – a file history (Map Editor)
