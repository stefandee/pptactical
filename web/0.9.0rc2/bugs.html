Fixed Bugs

  • Engine: firing all weapons won’t give any visual result; this was due to the CanBePlaced method which returned false for CIGDamage and CIGEffect objects :)
  • Corrected crashing bug in CIGBulletObject::Explode() if the effect sprite was null (in units editor editor there was a blank filename) – no filename, no sprite that is
  • Corrected a bug in Map Editor: at exit there was an exception; this was due to the fact that a global variable value (gPath, a string) was deallocated while the logfile calls still need the value
  • Saving a map in map editor may result in an exception: this was due to a wrong parameter passed to FMain::SaveMap
  • Taking screenshots resulted in corrupted bitmaps (Tamlin)
  • Hiding command panel (pressing space) had a small bug (Tamlin)
  • Right clicking in mini-map now sets the center of the view for the map (Tamlin)
  • Making teams did not worked (this was due the fact that at deserialization objects were
  • not added to OWNEDUNITS list)
  • Fixed a bug in FireAt script function which rendered the function useless
  • Fixed bugs when projectiles (rockets/bullets) exploding crashed


Known bugs

  • Objects may visualy dissapear ocassionaly from the map while moving
  • Moving the cursor over a unit while a group is selected makes a rectangle get cleared in the unit info area (it’s probably a forgotten control which is related to the mouse over action)
  • Unit info areas are in disorder (misaligned labels and such); that’s no problem, it is supposed to be this way
  • Units larger than 1x1 tiles behave at pathfinding like those of 1x1 size; this causes overlapping units. Will be fixed in future releases
